# Plan: Fix 403 Error & Add Error Handling

## Problem Summary
The frontend shows a 403 error when calling `GET /api/books/random`. This 403 originates from the **external Book Database API** and propagates through the backend to the frontend. The `.env` file contains `BOOKDATABASE_API_KEY`, but either:
1. Docker Compose isn't loading the `.env` (needs container rebuild), OR
2. The external API key is invalid/expired

## Root Cause
The backend's `BookDatabaseClient` doesn't handle external API failures. When `RestClient` throws `RestClientResponseException` (for 403/404/500), it bubbles up uncaught, resulting in poor UX and hard-to-diagnose errors.

## Solution Plan

### 1. Add External API Error Handling (Backend)
**File**: `backend/src/main/java/dev/rubenpari/backend/client/BookDatabaseClient.java`

- Wrap the `fetchRandom()` call with try-catch for `RestClientResponseException`
- On external API failure (4xx/5xx), throw a domain exception with context
- Log the actual error for debugging

**File**: `backend/src/main/java/dev/rubenpari/backend/controller/ApiExceptionHandler.java`

- Add handler for external API failures (return 502 Bad Gateway with user-friendly message)
- Add handler for generic exceptions (return 500 with safe message)

### 2. Ensure Env Vars Are Loaded (Ops Check)
User needs to verify Docker Compose is loading the `.env` file:
```bash
cd /Users/ruben/Desktop/books-randomized
docker-compose down
docker-compose up -d --build
```

Or explicitly:
```bash
docker-compose --env-file .env up -d --build
```

### 3. Add Favicon (Frontend)
**File**: Create `frontend/public/favicon.ico`

- Add a simple favicon to avoid the 404 noise in browser console
- Can use a placeholder book icon

## Implementation Details

### ApiExceptionHandler additions:
```java
@ExceptionHandler(RestClientResponseException.class)
public ResponseEntity<Map<String, String>> handleExternalApiError(RestClientResponseException ex) {
    // Log actual error for debugging
    logger.error("External API error: {} - {}", ex.getStatusCode(), ex.getResponseBodyAsString());
    // Return user-friendly 502
    return ResponseEntity.status(HttpStatus.BAD_GATEWAY)
        .body(Map.of("error", "Book database service unavailable. Please try again later."));
}

@ExceptionHandler(Exception.class)
public ResponseEntity<Map<String, String>> handleGeneric(Exception ex) {
    logger.error("Unexpected error", ex);
    return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR)
        .body(Map.of("error", "An unexpected error occurred"));
}
```

### BookDatabaseClient additions:
```java
public ExternalBook fetchRandom(Map<String, String> filters) {
    try {
        // existing code
    } catch (RestClientResponseException ex) {
        logger.error("BookDatabase API returned {}: {}", 
            ex.getStatusCode(), ex.getResponseBodyAsString());
        throw new IllegalStateException("Book database service error: " + ex.getStatusCode());
    }
}
```

## Benefits
1. **Better UX**: Users see "Book database service unavailable" instead of raw 403/500
2. **Easier debugging**: Backend logs show actual external API error
3. **Cleaner logs**: No more favicon 404 noise
4. **Resilience**: External API failures don't crash the app experience
